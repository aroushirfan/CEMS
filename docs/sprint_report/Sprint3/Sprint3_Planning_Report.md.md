# Sprint 3 Planning Report

### Duration: 11.02 → 03.02

### 1. Sprint Goal

- Extend the functional prototype with advanced features.

- Integrate Jenkins CI/CD pipeline for automated builds, tests, and code coverage.

- Develop and test a local Docker image of the project.

- Prepare the system for functional review, ensuring backend–frontend integration is stable.
### 2. Selected Product Backlog Items

- **US‑5 : Event notification** –Users receive notifications for approved or updated events.
- **US‑6: Attendance Tracking** – Users can register for events, and faculty and admin can track attendance .
- **US‑7: Profile management** – Users can update profile information.
- **US‑8: CI/CD integration** – Automate builds, tests, and code coverage with Jenkins.
- **US‑: Event Rsvp ** – Users can Rsvp to event .

### 3. Planned tasks

- Implement advanced features (event search, rsvp and track attendance).
- Configure Jenkins pipeline
- Generate JaCoCo code coverage reports
- Expand unit test coverage for new and existing features.
- Implement admin page and attendance tracking page for frontend .
- Connect frontend to backend with secondary testing
- Write unit tests.
- Prepare evidence for the sprint review 

### 4. Team capacity & Assumptions

It is agreed that the team will be able to complete the selected user stories within the sprint duration. The team will
also regulate communication and tracking of tasks using Trello.

### 5. Definition of done

- All selected user stories are implemented, tested, and functional.

- Jenkins CI/CD pipeline is operational with build, test, and coverage reports.

- Docker image runs successfully on local machine.

- Unit tests and coverage reports are up-to-date.

- Frontend and backend are fully integrated.

- Sprint review preparation is complete with evidence of team contributions.

- Trello/Jira boards reflect accurate completion of tasks.
