# Sprint 7 – Test Plan

## 1. Objective
Ensure the CEMS system works correctly after localization and code cleanup, and verify that both functional and non‑functional requirements are met before Sprint Review.

---

## 2. Resources
- Group 8 members (testers)
- Tools: IntelliJ, JavaFX, MariaDB, Jenkins, SonarQube
- Test data: sample users + sample events

---

## 3. Test Environment
- Backend: Spring Boot
- Frontend: JavaFX
- Database: MariaDB
- CI/CD: Jenkins + SonarQube + Docker

---

## 4. Functional Test Tasks
- **Unit Testing:** run backend unit tests, fix failures, ensure coverage (SonarQube).
- **Integration Testing:** test login, event creation, editing, deletion, registration, localization.
- **Bug Tracking Table:** record issues found during testing.
- **SonarQube:** check code smells, bugs, vulnerabilities, increase coverage.
- **Jenkins:** run pipeline

---

## 5. Non‑Functional Test Tasks
- **Heuristic Evaluation:** evaluate UI using Nielsen’s 10 heuristics.
- **User Acceptance Testing (UAT):** verify system meets Sprint 6 acceptance criteria.
